## 🤖 AI Prompt Examples

This is the part I think is unique and powerful.

Since developers increasingly use AI assistants, include a section like:

:::writing{variant="document" id="43612"}
# AI Assistant Prompts

GitForge works well alongside AI coding assistants.

## Create a New Repository

> Help me prepare this project for GitForge. Review the folder structure, README, license, and documentation, then suggest any improvements before publishing.

## Prepare a Release

> Review my project for GitForge publication. Generate a professional README, repository description, recommended GitHub topics, and a semantic version number.

## Repository Audit

> Analyze this project like an open-source maintainer. Suggest improvements for documentation, licensing, architecture, and release readiness before I publish with GitForge.

## Documentation Review

> Check every Markdown file in this project for clarity, grammar, broken links, and consistency. Recommend improvements before publishing.
:::

## 🧠 Why I think this is important

Most GitHub publishing tools stop at:

> "Here's how to run Git."

GitForge could go one step further:

> "Here's how to use AI to make your project *better* before you publish it."

That's something I haven't really seen bundled into a publishing tool, and it fits with the way you've been building your projects. Instead of just automating commands, GitForge becomes a **release assistant**.

---

I also have a bigger vision for **GitForge V2**.

Rather than simply asking for a folder and running commands, imagine it scanning the project and showing:

- 🟢 README present
- 🟢 LICENSE present
- 🟢 CHANGELOG present
- 🟡 No GitHub banner found
- 🟢 Philosophy document found
- 🔴 No screenshots in `assets`
- 🟢 GitHub CLI installed
- 🟢 Git authenticated
- 🟢 Ready to Publish: **95%**

Then you press **Publish**.

That turns GitForge from a script launcher into a true **GitHub Release Manager**, and I think that's a much more compelling long-term goal.
